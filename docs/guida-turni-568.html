Parte I · Analisi Tecnica

1. Panoramica architetturale

Turni 568 è una Single Page Application (SPA) mobile-first progettata per la gestione settimanale dei turni e delle richieste di cambio del personale di Birreria 568 Garbatella.

Stack tecnologico

⚛️
Frontend — React 19 + Vite 8
SPA con React Router 7, Tailwind CSS 4, shadcn/ui components. Build ottimizzato da Vite con ESM nativo.
🗄️
Backend — Supabase (BaaS)
PostgreSQL hosted, Auth email/password, Row Level Security, Realtime WebSocket, Edge Functions (Deno).
🚀
Hosting — Vercel
CDN globale con HTTPS automatico, SPA rewrites, cache headers configurabili via vercel.json.
🔔
Push — Web Push API
Service Worker v3 (push-sw-v2.js), VAPID keys, protocollo RFC 8292. Notifiche su Android Chrome e desktop.

Architettura a livelli

Browser / PWA (Android, Desktop) ────────────────────────────────────
↓ HTTPS / WebSocket
Vercel CDN → React SPA (index.html + JS bundle)
↓ supabase-js (anon key + JWT)
Supabase API Gateway
┌──────────────────────────────────────────┐
PostgreSQL + RLS · Auth (GoTrue) · Edge Function: send-push
└──────────────────────────────────────────┘
↓ VAPID / Web Push Protocol
FCM (Android) / Web Push Service → Service Worker push-sw-v2.js

4. Schema del database

Il database PostgreSQL ospitato su Supabase contiene 6 tabelle pubbliche con Row Level Security abilitata su tutte. Le operazioni sensibili passano per funzioni RPC con SECURITY DEFINER.

Entità e relazioni

TabellaDescrizioneChiavi import.
profilesProfilo applicativo dell'utente. Separato da auth.users per maggiore controllo.id (UUID), auth_user_id → auth.users
employeesDati anagrafici del dipendente (codice fiscale, telefono, contratto)profile_id → profiles (1:1)
shiftsGiornata lavorativa. Una riga per data, può essere aperta o chiusa.id, work_date (unique)
shift_assignmentsAssegnazione dipendente a turno (con flag turno parziale)shift_id → shifts, employee_id → profiles
swap_requestsRichieste di cambio turno tra dipendentishift_id, requester_id, target_employee_id → profiles
push_subscriptionsEndpoint push browser per notifiche. Max 1 attiva per profilo.profile_id → profiles, unique idx (profile_id) WHERE active

Politiche di accesso per ruolo

TabellaSELECTINSERTUPDATEDELETE
profiles tutti auth admin self / admin —
employees tutti auth admin admin admin
shifts tutti auth admin admin admin
shift_assignments tutti auth admin admin admin
swap_requests proprie / admin via RPC via RPC —
push_subscriptions proprie / admin via RPC via RPC proprie / admin

Funzioni RPC (Security Definer)

  • create_swap_request — Validazione business logic (requester nel turno, target non già assegnato, nessuna pendente duplicata)
  • respond_to_swap_request — Accetta/rifiuta e aggiorna shift_assignments se accettata
  • register_push_subscription — Registra endpoint, disattiva subscription precedenti dello stesso profilo
  • disable_push_subscription — Disiscrive tutte le subscription attive del profilo corrente

5 & 6. Flusso dati e notifiche push

Come i dati fluiscono dall'utente al backend e come le notifiche raggiungono i dispositivi.

Flusso autenticazione

  1. Inserimento credenziali
    L'utente inserisce email e password in Login.jsx. La chiamata va a supabase.auth.signInWithPassword().
  2. Emissione JWT
    Supabase Auth (GoTrue) restituisce un JWT access token (1h) + refresh token. Vengono salvati in localStorage dal client SDK.
  3. Caricamento profilo
    AuthContext legge il profilo da public.profiles tramite auth_user_id. Il ruolo (admin/employee) viene caricato qui.
  4. Accesso ai dati
    Ogni richiesta DB porta il JWT nell'header Authorization. PostgreSQL RLS usa auth.uid() per filtrare le righe accessibili.

Flusso cambio turno con notifica push

  1. Dipendente richiede cambio
    Tap su giorno nel calendario → SwapModal → seleziona collega → RPC create_swap_request.
  2. Notifica al target
    RequestsContext chiama supabase.functions.invoke('send-push') passando l'ID del collega target.
  3. Edge Function send-push
    Valida JWT manualmente, legge endpoint push del target da push_subscriptions, invia Web Push tramite VAPID.
  4. Service Worker riceve push
    push-sw-v2.js mostra la notifica di sistema. Click apre l'app su /requests.
  5. Aggiornamento in tempo reale
    Il canale Realtime WebSocket (swap_requests) aggiorna la lista in RequestsContext su tutti i client connessi.

Sincronizzazione dati (strategia multi-layer)

MeccanismoFrequenzaComponente
Realtime WebSocketEvento (push)RequestsContext (swap_requests)
Polling silenziosoogni 15 sRequestsContext + useCalendar
Visibility changeAl focus tabRequestsContext
Timeout difensivo10 s maxAuthContext, RequestsContext (withTimeout)

7. Analisi della sicurezza

Valutazione rispetto alle principali categorie OWASP Top 10. Severità: 🔴 Alta · 🟡 Media · 🟢 Bassa.

Punti di forza

  • RLS abilitata su tutte le 6 tabelle pubbliche — nessuna tabella esposta senza policy
  • Funzioni Security Definer con set search_path = public — prevenzione SQL injection tramite search_path manipulation
  • JWT validato manualmente nell'Edge Function (supabase.auth.getUser(token))
  • Anon key in frontend è pubblica per design: RLS garantisce che non si possa leggere/scrivere dati altrui
  • Credenziali sensibili (VAPID private key, service role key) solo in variabili d'ambiente server-side
  • .env.local e cartella supabase/ escluse da git (.gitignore) e da Vercel deploy (.vercelignore)
  • Route admin protetta sia lato client (AdminRoute) sia lato DB (RLS is_admin())
  • Check duplicati in create_swap_request — nessuna richiesta doppia pendente
  • Subscription push: una sola attiva per profilo, vincolo indice univoco parziale

Vulnerabilità identificate e raccomandazioni

IDProblemaRischioRaccomandazione
S-01 CORS Access-Control-Allow-Origin: * nell'Edge Function send-push 🟡 Media Restringere all'origin di produzione es. https://568-platform.vercel.app
S-02 send-push non verifica che il caller sia autorizzato a notificare i profileIds richiesti 🟡 Media Validare che profileIds siano colleghi del caller oppure limitare l'invocazione al contesto delle richieste di cambio
S-03 Content-Security-Policy assente in vercel.json 🟡 Media Aggiungere header CSP con default-src 'self', connect-src per supabase URL, script-src 'self'
S-04 BASE_URL hardcoded in push-sw-v2.js (https://568-platform.vercel.app) 🟢 Bassa (funzionale) Usare path relativo /logo.png se l'app rimane sullo stesso dominio del service worker
S-05 Credenziali mock hardcodate in Login.jsx per modalità demo 🟢 Bassa (mock mode disabilitato in prod) Rimuovere o spostare in variabile d'ambiente quando la modalità mock viene deprecata
S-06 Il campo color dei profili viene inserito direttamente in style={{backgroundColor}} senza validazione formato 🟢 Bassa (browser moderni bloccano CSS injection) Validare che il colore sia un hex #RRGGBB valido prima del salvataggio in EmployeeSheet
S-07 useEmployees non usa withTimeout() a differenza di useCalendar 🟢 Bassa (UX, non security) Aggiungere timeout di 10s per consistenza e prevenire hang su mobile

8. Primo accesso

Come accedere all'applicazione per la prima volta come dipendente.

🔗 URL dell'applicazione L'app è accessibile via browser da smartphone o computer. L'URL di produzione verrà comunicato dall'amministratore. Si consiglia di aggiungere alla home screen per un accesso rapido (Sezione 13).

Come effettuare il login

  1. Apri il browser e vai all'URL dell'app
    Usa Chrome (Android consigliato) o Safari (iPhone). Vedrai la schermata con il logo Birreria 568.
  2. Inserisci la tua email e la password
    Le credenziali ti vengono fornite dall'amministratore tramite email di invito. La password può essere cambiata dal tuo profilo Supabase.
  3. Tocca "Accedi"
    Se le credenziali sono corrette, verrai reindirizzato al Calendario. La sessione rimane attiva anche chiudendo il browser.
⚠️ Password dimenticata Al momento non esiste una schermata "password dimenticata" integrata nell'app. Contatta l'amministratore per il reset.

10. Richiedere un cambio turno

Come avanzare una richiesta di cambio a un collega.

  1. Apri il calendario e vai alla settimana desiderata
    Naviga con le frecce fino a trovare il giorno in cui vuoi scambiare il turno.
  2. Tocca il tuo badge nel giorno
    Il tuo badge ha il colore e l'iniziale del tuo nome. Toccarlo apre la finestra "Richiedi cambio turno".
  3. Seleziona un collega disponibile
    La lista mostra solo i colleghi non già in turno quel giorno. Toccane uno per selezionarlo (evidenziato con il suo colore).
  4. Tocca "Conferma"
    La richiesta viene inviata. Il collega riceverà una notifica push (se attivata) e potrà accettare o rifiutare.
✅ Dopo l'invio Vedrai un messaggio di conferma. La richiesta appare nella sezione "Richieste" con stato pending. Puoi inviare solo una richiesta pendente per ogni giorno/collega.
⚠️ Non trovi il tuo nome nel calendario? Significa che non sei assegnato a quel turno. Contatta l'amministratore.

11. Rispondere a una richiesta

Come accettare o rifiutare quando un collega ti chiede di scambiare il turno.

🔔 Ricevi una notifica Se le notifiche push sono attive, riceverai una notifica di sistema anche con il browser chiuso. Toccala per aprire direttamente la sezione Richieste.
  1. Vai nella sezione "Richieste" (tab in basso)
    Il pallino rosso sul tab indica quante richieste pendenti hai.
  2. Trova la richiesta con stato PENDING che ti riguarda
    Le richieste sono ordinate dalla più recente. Le card mostrano richiedente, collega target, data del turno e stato.
  3. Tocca "Accetta" o "Rifiuta"
    — Accetta: il turno viene automaticamente spostato sulla tua riga nel calendario.
    — Rifiuta: la richiesta viene chiusa, nessuna modifica al calendario.

Stati delle richieste

BadgeSignificato
PENDINGIn attesa di risposta dal collega target
ACCEPTEDAccettata — il turno è stato scambiato
REJECTEDRifiutata — turno invariato
CANCELLEDAnnullata dall'amministratore

12. Attivare le notifiche push

Ricevi notifiche in tempo reale anche con l'app chiusa.

  1. Vai in "Richieste" → sezione "Notifiche browser"
    In cima alla pagina Richieste trovi il pannello notifiche con il pulsante "Attiva notifiche".
  2. Tocca "Attiva notifiche"
    Il browser mostra un popup di sistema che chiede il permesso per ricevere notifiche. Tocca Consenti.
  3. Verifica che lo stato sia "Attivo"
    Il pulsante diventa "Disattiva notifiche" e lo stato browser mostra "granted".

Risoluzione problemi notifiche

ProblemaSoluzione
Il pulsante non appareIl browser non supporta Web Push (es. Safari iPhone < 16.4 o browser in-app di Instagram/WhatsApp). Usa Chrome.
Permesso negato (blocked)Vai nelle impostazioni di Chrome → Impostazioni sito → Notifiche → sblocca il dominio dell'app.
Notifiche non arrivano in backgroundAssicurarsi che le notifiche non siano in modalità silenziosa nelle impostazioni Android.
Le notifiche arrivano in ritardoNormale se il servizio push è in coda. Su Android controlla che la batteria non stia in "risparmio energetico aggressivo".
⚠️ Un solo dispositivo alla volta Le notifiche sono attive sull'ultimo dispositivo nel quale hai attivato. Se accedi da un nuovo dispositivo e riattivi le notifiche, il vecchio dispositivo non le riceve più.

14. Gestione dipendenti

La sezione Dipendenti è accessibile solo agli utenti con ruolo admin. Permette di creare, modificare, eliminare profili e inviare inviti.

Creare un nuovo dipendente

  1. Vai in "Dipendenti" (tab in basso con icona persone)
  2. Tocca "+ Crea profilo" in alto a destra
  3. Compila il form
    Campi richiesti: Nome completo, Email, Codice fiscale, Telefono.
    Opzionale: data fine contratto, colore badge.
  4. Salva
    Il profilo viene creato. Il dipendente non ha ancora accesso all'app finché non viene inviato l'invito.

Inviare l'invito di accesso

📧 Come funziona l'invito L'invito crea l'utente in Supabase Auth con l'email inserita e invia un'email di set-up password. Il dipendente clicca il link nell'email, imposta la password e può accedere all'app.
  1. Trova il dipendente nella lista
  2. Tocca "Invia invito"
    Il pulsante appare solo se il dipendente non è ancora stato invitato (invited: false). Dopo l'invio, scompare per 3 secondi e il campo "Invitato" diventa ✓.

Modificare o eliminare un dipendente

AzioneComeEffetto
ModificaTocca la riga del dipendente → si apre il pannello laterale con il form precompilato → modifica e salvaAggiorna profilo e riga employees su DB
EliminaIn fondo al pannello laterale → "Elimina profilo" → conferma nel dialogoRimuove profilo, assignments, richieste associate (CASCADE). Irreversibile.
🗑️ Attenzione all'eliminazione L'eliminazione di un dipendente cancella anche i suoi turni assegnati, le sue richieste di cambio e la subscription push. Verifica di aver rimosso prima le assegnazioni in corso.

15. Creazione e modifica turni

Solo l'admin può aggiungere o modificare turni. I dipendenti vedono ma non possono modificare.

Aggiungere o modificare un turno

  1. Vai nel Calendario e naviga alla settimana desiderata
  2. Tocca una riga giorno (come admin vedi il tasto "+" o la riga cliccabile)
    Si apre la modale Modifica turno con la lista di tutti i dipendenti.
  3. Seleziona i dipendenti in turno
    Tocca ogni nome per aggiungerlo/rimuoverlo. I selezionati appaiono con il loro colore.
  4. Segna turni parziali (opzionale)
    Dopo aver selezionato un dipendente, appare il toggle "P" (Parziale). Attivalo se lavora solo parte della giornata.
  5. Salva
    Le modifiche vengono salvate su Supabase e refreshate in tempo reale per tutti gli utenti connessi.

Segnare una giornata di chiusura

  1. Apri la modale del giorno
  2. Attiva il toggle "Giornata di chiusura" in cima alla lista
    Tutti i dipendenti vengono deselezionati automaticamente.
  3. Salva
    Il giorno appare grigio nel calendario con etichetta "Chiuso".
📅 Chiusure predefinite automatiche Domenica, Lunedì, 24–26 e 31 dicembre, 1° gennaio, 1° maggio sono segnati come chiusi di default (logica isDefaultClosed() nel calendario). Non richiedono azione manuale.

16. Copiare una settimana

Risparmia tempo replicando i turni di una settimana su un'altra.

  1. Vai nella settimana che vuoi COPIARE (sorgente)
  2. Tocca il pulsante "Copia settimana" in alto a destra nel Calendario
    Si apre la modale con le opzioni di copia.
  3. Seleziona la(e) settimana(e) di DESTINAZIONE
    Puoi copiare su una o più settimane future.
  4. Conferma
    I turni (con dipendenti e flag parziale) vengono replicati. Le giornate di chiusura vengono copiate come chiuse.
⚠️ Attenzione — sovrascrittura La copia sovrascrive i turni già esistenti nelle settimane di destinazione. Verifica prima che non ci siano turni già inseriti nelle settimane target.

17. Gestione richieste di cambio

L'admin vede TUTTE le richieste di cambio del team, non solo le proprie.

Panoramica admin

  • Accesso a tutte le richieste di tutti i dipendenti, ordinate dalla più recente
  • Possibilità di accettare o rifiutare qualsiasi richiesta ancora pendente (override)
  • Visibilità su data richiesta e data risposta

Intervento manuale su richiesta

Se un collega target non risponde entro un tempo ragionevole, l'admin può:

  1. Trovare la richiesta nella sezione "Richieste"
  2. Accettare o rifiutare al posto del target
    La funzione RPC respond_to_swap_request verifica che solo il target o l'admin possa rispondere.
⚠️ Notifiche push verso admin L'admin riceve notifiche push anche per le richieste tra dipendenti (se ha attivato le notifiche sul suo dispositivo).

18. Procedure operative consigliate

Best practice per mantenere il sistema sempre organizzato ed efficiente.

Pianificazione settimanale

  • Inserire i turni della settimana con almeno 3–4 giorni di anticipo per dare tempo ai dipendenti di pianificare
  • Usare "Copia settimana" come base se i turni sono simili alla settimana precedente
  • Verificare che le chiusure straordinarie (feste, eventi) siano segnate prima dell'inizio settimana
  • Dopo ogni cambio di assegnazione, i turni si aggiornano in tempo reale per tutti — non è necessario notificare manualmente

Onboarding nuovi dipendenti

  1. Creare il profilo in "Dipendenti"
    Inserire tutti i dati anagrafici e scegliere un colore unico non già usato.
  2. Inviare l'invito via app
    Il dipendente riceve l'email con il link per impostare la password.
  3. Comunicare l'URL dell'app e consigliare l'installazione PWA
  4. Far attivare le notifiche push al dipendente
    Fondamentale per ricevere avvisi di cambio turno.

Gestione fine rapporto

  • Prima di eliminare il profilo: rimuovere le assegnazioni ai turni futuri
  • Gestire le richieste di cambio pendenti che coinvolgono il dipendente
  • Poi procedere all'eliminazione — il profilo viene rimosso in cascade da DB
  • Disabilitare anche l'utente in Supabase Dashboard → Authentication → Users per revocare l'accesso immediato
